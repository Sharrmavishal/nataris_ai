# Contributing to Nataris

Thank you for your interest in contributing to Nataris! This document provides guidelines for contributing to our public repositories.

## Ways to Contribute

### 1. Report Issues
- **Bug reports:** Found something broken? Open an issue with steps to reproduce
- **Feature requests:** Have an idea? We'd love to hear it
- **Documentation:** Spotted a typo or unclear docs? Let us know

### 2. Improve Documentation
- Fix typos and grammatical errors
- Add examples and clarifications
- Translate documentation

### 3. Share Examples
- Create integration examples in your favorite language/framework
- Share use cases and tutorials
- Build demo applications

### 4. Provide Feedback
- Test new features and provide feedback
- Participate in discussions
- Help other community members

## Code of Conduct

### Our Standards

- **Be respectful:** Treat everyone with respect and kindness
- **Be constructive:** Provide helpful feedback and suggestions
- **Be inclusive:** Welcome newcomers and diverse perspectives
- **Be patient:** Remember that everyone is learning

### Unacceptable Behavior

- Harassment, discrimination, or offensive language
- Personal attacks or trolling
- Spam or self-promotion
- Sharing others' private information

## Submitting Contributions

### For Documentation/Examples

1. Fork this repository
2. Create a branch (`git checkout -b improve-docs`)
3. Make your changes
4. Test your changes locally
5. Commit with a clear message (`git commit -m "docs: add Python example"`)
6. Push to your fork (`git push origin improve-docs`)
7. Open a Pull Request

### Commit Message Format

We follow conventional commits:

```
type(scope): description

[optional body]
```

Types:
- `docs`: Documentation changes
- `examples`: New or updated examples
- `fix`: Bug fixes
- `feat`: New features
- `chore`: Maintenance tasks

### Pull Request Guidelines

- Keep changes focused and atomic
- Include a clear description of what and why
- Link related issues if applicable
- Ensure all tests pass (if applicable)
